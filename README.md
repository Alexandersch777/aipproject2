# Тренажёр по матричной алгебре

Консольное приложение на C++ для тренировки матричных вычислений. Программа
генерирует задания со случайными матрицами, гарантируя **целочисленные ответы**,
и проверяет введённое пользователем решение.

## Возможности

Пользователь выбирает уровень сложности (он задаёт размер матриц) и тип задания:

- Умножение — перемножение двух матриц (не обязательно квадратных).
- Определитель — вычисление определителя квадратной матрицы.
- Метод Гаусса — решение системы A·x = b.
- Обратная матрица — нахождение обратной матрицы.

Уровни сложности: лёгкий (2), средний (3), сложный (4).

Целочисленность ответов обеспечивается так:
- **определитель** целой матрицы всегда целый;
- для **Гаусса** и **обратной** матрица строится унимодулярной (определитель `±1`)
  с помощью элементарных преобразований строк, поэтому решение и обратная — целые.

## Сборка

Требуется CMake (≥ 3.16) и компилятор C++17. Зависимость для тестов (doctest)
скачивается и собирается автоматически через `FetchContent`.

```sh
cmake -S . -B build
cmake --build build
```

## Запуск

```sh
./build/matrix_trainer
```

Исполняемый файл самодостаточен (рантайм слинкован статически) — его можно
запустить и двойным кликом.

Необязательные аргументы (пути задаются пользователем, разделитель `/`,
абсолютные пути не зашиты в код):

```sh
./build/matrix_trainer --seed 42 --log logs/session.log
```

- `--seed N` — фиксированное зерно генератора (для воспроизводимости);
- `--log путь` — файл, в который дописывается результат каждого раунда.

## Тесты

```sh
cd build
ctest --output-on-failure
```

Тесты написаны на **doctest**, собираются из той же библиотеки `matrix_core`,
что и приложение, и запускаются командой `ctest`.

## Структура проекта

```
include/matrix/   заголовки с документацией в стиле Doxygen
src/              реализация (Matrix, Generator, Exercise) и консольный интерфейс
tests/            тесты на doctest
CMakeLists.txt    сборка приложения и тестов
.clang-format     стиль форматирования (Google, отступ 4)
```

## Соответствие требованиям

- Сборка через **CMake**, запуск из консоли и кликом.
- Нет прямого управления памятью (`new`/`delete`/`malloc` отсутствуют).
- Все ошибки — через **исключения** (`MatrixError`, `InputError`), обработаны в `main`.
- Сторонняя библиотека (doctest) собирается **CMake** (`FetchContent`).
- Нет абсолютных путей; разделитель `/`; пути задаются пользователем.
- Кодировка файлов — **UTF-8**; код отформатирован по **.clang-format**.
- Документация в формате **Doxygen** в заголовочных файлах.
- Более трёх функций; тесты покрывают положительные и отрицательные исходы.
